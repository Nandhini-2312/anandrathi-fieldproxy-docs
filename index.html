Anand Rathi × FieldProxy — Integration Guide

One place for the database schema, REST endpoints and ready-to-paste sample requests for the ARSSBL ↔ FieldProxy integration: daily employee attendance (punch-in / punch-out with selfie + GPS), the user master synced from the E-HR system, and the team master.

v1.0 · 2026 Tenant · arssbl Base URL · https://api-india-1.fieldproxy.ai Auth · x-api-key (JWT)

Overview

FieldProxy is the operational database for ARSSBL's field workforce. Three tables are in scope:

  • attendance_ukrwi — one row per employee per working day. Records punch-in / punch-out selfie, GPS coordinates, time, address, and computed working hours.
  • fp_users — master list of users who can log in to the FieldProxy mobile / web app. ARSSBL pushes this whenever a user is created, updated, or deactivated in the E-HR system.
  • fp_teams — master list of teams / groups that users are assigned to. Each team links to fp_users.teamid.

Every change is a single REST call against FieldProxy's /api/insert, /api/update, /api/read, or /api/delete endpoint with an x-api-key header.

Database schema overview

Three tables. fp_users.teamid links to fp_teams.team_id; attendance_ukrwi.employee matches fp_users.userid.

fp_teams                         (team master)
   └─ fp_users                   (user master · teamid -> fp_teams.team_id)
        └─ attendance_ukrwi      (daily attendance · employee -> fp_users.userid)

B · Update an existing user

curl --request POST 'https://api-india-1.fieldproxy.ai/api/update' \
  --header 'x-api-key: <YOUR_API_KEY>' \
  --header 'Content-Type: application/json' \
  --data '{
  "tableName": "fp_users",
  "data": { "username": "Updated Name", "email": "updated@example.com" },
  "condition": { "userid": "EMP10234" }
}'
